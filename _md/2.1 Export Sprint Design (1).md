## 2\. Excel File Design

### 2.1 Components:

* Excel Template File: [sprint.xlsx](https://docs.google.com/spreadsheets/d/18pk_EFk9VSQI0Uxo1mfSxZGlzIKXUks9/edit?usp=drive_link&ouid=118226940259211287797&rtpof=true&sd=true)  
* Phân tích:  
  * Table 1: Sprint  
    * Giới thiệu về sprint (name, start/end date)  
  * Table 2: Member  
    * List members, và thống kê story point làm được (expected/actual)  
  * Table 3: List work  
    * Hiển thị các tasks trong sprint, tiến độ của các task đó (theo story point và thời gian trong khoảng sprint)  
    * Thống kê lại thời gian thực tế và thời gian ước tính  
  * Table 4:  
    * Bảng metrics, Hiển thị các spec: Total estimate, total completed, completion rate, spillover, unestimated count  
* SRS: [Export Srpint Requirement Analysis](https://docs.google.com/document/d/141QbNV5nVr0AL2K-ubW94nJc1HEp7YKX3Ew1ESSIAAw/edit?tab=t.0#heading=h.latn9x1vwhpt)

### 2.2 Input & Output Analysis:

1) Input: [1\. LLD Phase 1](https://docs.google.com/document/d/10-Xr_55o0Xi4ooyNlKYYBgp_HDCke96_mvrp2QoXdWo/edit?tab=t.0#heading=h.h4e59of8ypi8)  
* Database & Entity in LLD Phase 1

2) Output  
* File Excel với một số value:  
  * Table 2: Member  
    * Actual Field: Sum(tasks-story point-actual, if member \= task\_member)  
    * Expected Field: Sum(tasks-story point-expected, if member \= task\_member)  
  * Table 3: Tasks  
    * Task statistic  
      * Expected total: Sum(task expected)  
      * Actual total: Sum (task actual)  
      * Daily expected array: Expected\[i\] \= Sum(daily task expected)  
      * Daily actual array: Actual\[i\] \= Sum(daily task actual)  
    * Task data:  
      * Field: Task name, member, completed at, actual point, expected point: From input  
      * Daily cell value per task: Calculate from completed at (if the completed at is that day \=\> fill 0 \-\> Done, otherwise keep it expected point \-\> Not Done)  
      * Daily cell color per task:   
        * Nếu completed\_at \<= day → \= 0  
        * Nếu completed\_at \> day → \= story\_point  
      *  Without story point: Expected: UNESTIMATED, Actual: empty  
      * Not done in sprint: keep the story point in full date range (eg. Task A not done with 2 point, so from day 1 \-\> 10, each day cell value \= 2, and no decrease)  
    * Metric  
      * Total Estimate: Sum(tasks\_story\_point, if story point \!= nil)  
      * Total Completed: Sum(tasks\_story\_point, if: story point \!= nil && completed at \!= nil)  
      * Completion Rate: (Total Completed / Total Estimate)\*100  
      * Unestimated Count: Sum(tasks number, if story point \== nil)  
    * Burndow Table  
      * Deriving from the task table (no calculation here, only get from above)
